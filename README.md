# System Design Journey

I created this repository to document my learning journey in system design, from the absolute basics to interview-level thinking. Instead of trying to memorize architectures, I want to understand how systems actually work, why certain components exist, and what tradeoffs engineers make as systems grow.

Each lesson in this repo focuses on one concept at a time and is paired with practical examples or small projects. My goal is to learn in public, keep things beginner-friendly, and build a stronger understanding of system design step by step.
